# Посібник з розгортання в Production (Release Engineer / DevOps)

Цей документ містить інструкції для розгортання React-додатку (Vite) у виробничому середовищі. Наш проект є статичним фронтенд-додатком, що потребує лише веб-сервера для роздачі контенту.

---

## 1. Вимоги до апаратного забезпечення

Оскільки додаток є статичним і виконується в браузері клієнта, вимоги до сервера мінімальні:

- **Архітектура**: x86_64 або ARM64.
- **CPU**: 1 ядро (мінімум), 2 ядра (рекомендовано для збірки на сервері).
- **Пам'ять (RAM)**: 512 МБ (мінімум), 1 ГБ або більше (якщо збірка `npm run build` виконується безпосередньо на сервері).
- **Диск**: ~100 МБ для статичних файлів + місце для кешу та логів веб-сервера.

---

## 2. Необхідне програмне забезпечення

Для успішного розгортання на сервері повинні бути встановлені:

1.  **Node.js (LTS)** та **npm**: для збірки проекту (якщо не використовується збірка в CI/CD).
2.  **Веб-сервер**: Nginx (рекомендовано), Apache або Caddy.
3.  **Система контролю версій**: Git.
4.  **SSL-сертифікат**: Certbot (Let's Encrypt) для підтримки HTTPS.

---

## 3. Налаштування мережі

- **Відкриті порти**:
    - `80` (HTTP) — для редіректу на HTTPS.
    - `443` (HTTPS) — основний порт для доступу до додатку.
- **DNS**: Має бути запис `A` або `CNAME`, що вказує на IP-адресу сервера.

---

## 4. Конфігурація веб-сервера (Nginx)

Приклад мінімальної конфігурації для роботи SPA (Single Page Application):

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/react-app/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Кешування статичних активів
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, no-transform";
    }
}
```

*Важливо: `try_files` забезпечує коректну роботу роутингу на стороні клієнта.*

---

## 5. Налаштування СУБД

**Цей проект не має власної бази даних на сервері.** Якщо додаток використовує зовнішні API, переконайтеся, що:
- Домен додатку доданий до списку дозволених у налаштуваннях CORS бекенд-сервісів.
- Всі змінні середовища (env) налаштовані правильно.

---

## 6. Розгортання коду

### Рекомендований шлях (CI/CD):
1. Код збирається в CI (GitHub Actions, GitLab CI тощо) за допомогою `npm run build`.
2. Вміст папки `dist/` передається на сервер через `scp` або за допомогою Docker.

### Ручне розгортання:
```bash
git clone <URL_репозиторію>
cd react-app
npm install
npm run build
cp -r dist/* /var/www/react-app/dist/
```

---

## 7. Перевірка працездатності (Health Check)

Щоб переконатися, що додаток працює правильно:

1.  **HTTP Статус**: Виконайте `curl -I https://your-domain.com`. Ви повинні отримати статус `200 OK`.
2.  **Завантаження контенту**: Відкрийте сайт у браузері та перевірте 콘соль розробника (F12) на наявність помилок 404 або помилок скриптів.
3.  **SPA Routing**: Перейдіть за прямим посиланням на будь-який внутрішній маршрут (наприклад, `/about`) і оновіть сторінку. Якщо сервер повертає `index.html`, налаштування `try_files` працює правильно.
4.  **SSL**: Перевірте термін дії сертифіката через браузер (замочок у адресному рядку).
