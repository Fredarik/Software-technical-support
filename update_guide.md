# Посібник з оновлення та відкату (Release Engineer / DevOps)

Цей документ містить покрокові інструкції щодо оновлення проекту у виробничому середовищі та процедуру повернення до попередньої версії (rollback).

---

## 1. Підготовка до оновлення

### Створення резервних копій (Backup)
Перед початком оновлення обов'язково створіть копію поточної стабільної версії:
- Копіюйте директорію зі статичними файлами (`dist/` або `/var/www/react-app/dist`) в окремий архів з відміткою часу.
- Якщо використовуються специфічні `env` файли, збережіть їх копії.

### Перевірка сумісності
- Переконайтеся, що версія Node.js на сервері відповідає вимогам нової збірки (`package.json`).
- Перевірте, чи не потребує нова версія оновлення конфігурації веб-сервера (Nginx).

### Планування часу простою (Downtime)
Для статичних React-додатків час простою зазвичай дорівнює **0 секунд**, оскільки оновлення відбувається шляхом заміни файлів. Однак рекомендується проводити оновлення в години найменшої активності користувачів.

---

## 2. Процес оновлення

### Крок 1: Зупинка служб
Для роздачі статичних файлів зупиняти Nginx **не потрібно**. Веб-сервер продовжує обслуговувати старі файли з пам'яті, доки не будуть підтягнуті нові.

### Крок 2: Розгортання нового коду
1. Отримайте нову версію коду (`git pull` або завантаження артефакту).
2. Виконайте збірку:
   ```bash
   npm install
   npm run build
   ```
3. Замініть вміст робочої папки:
   ```bash
   # Рекомендовано: використовувати атомарну заміну через symlink
   # Або просто копіювання:
   cp -r dist/* /var/www/react-app/dist/
   ```

### Крок 3: Міграція даних та конфігурацій
- Оскільки це фронтенд-додаток, міграції БД на цьому рівні відсутні.
- Перевірте файл `.env.production`. Якщо додалися нові ключі API, додайте їх до конфігурації сервера.

---

## 3. Перевірка після оновлення

1. Очистіть кеш браузера (або відкрийте вкладку інкогніто) та завантажте сайт.
2. Перевірте версію додатка (якщо вона відображається в футері або консолі).
3. Переконайтеся, що всі запити до API повертають статус `200` або `201`.

---

## 4. Процедура відкату (Rollback)  (+1 бал)

Якщо під час перевірки виявлено критичні помилки, виконайте наступні дії для миттєвого відновлення працездатності:

1.  **Повернення файлів**: Замініть пошкоджену версію файлами з резервної копії, створеної на етапі підготовки:
    ```bash
    rm -rf /var/www/react-app/dist/*
    cp -r /var/www/backups/dist_20260327_stable/* /var/www/react-app/dist/
    ```
2.  **Повернення конфігурацій**: Якщо ви змінювали `.env` або конфіги Nginx, відновіть їх з бекапів.
3.  **Перезапуск сервера** (якщо змінювався конфіг):
    ```bash
    sudo nginx -t && sudo systemctl reload nginx
    ```
4.  **Аналіз**: Після відновлення стабільної версії проаналізуйте логи (`/var/log/nginx/error.log`), щоб знайти причину невдалого оновлення.
