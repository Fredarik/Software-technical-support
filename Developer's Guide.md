# React App — Посібник розробника

Ласкаво просимо до нашого проекту! Цей документ допоможе вам налаштувати середовище розробки та запустити проект на "чистій" операційній системі.

---

## 1. Необхідне програмне забезпечення

Перед початком роботи переконайтеся, що на вашому комп'ютері встановлено:

1.  **Git**: [Завантажити Git](https://git-scm.com/downloads) (для клонування репозиторію).
2.  **Node.js**: [Завантажити Node.js](https://nodejs.org/) (рекомендовано версію **LTS**).
    *   Разом з Node.js автоматично встановиться пакетний менеджер **npm**.
3.  **Visual Studio Code (VS Code)**: [Завантажити VS Code](https://code.visualstudio.com/) (рекомендований редактор).

---

## 2. Налаштування середовища

### Крок 1: Клонування репозиторію
Відкрийте термінал (або PowerShell на Windows) і виконайте команду:
```bash
git clone <URL_репозиторію>
cd react-app
```

### Крок 2: Встановлення залежностей
У кореневій папці проекту виконайте команду для завантаження всіх необхідних бібліотек:
```bash
npm install
```

---

## 3. Налаштування бази даних
**Цей проект не потребує локальної бази даних.** Дані обробляються на фронтенді або через зовнішні API. Якщо в майбутньому знадобиться БД, інструкції будуть додані сюди.

---

## 4. Запуск проекту у режимі розробки

Щоб запустити проект локально з підтримкою "гарячого оновлення" (HMR), виконайте:
```bash
npm run dev
```
Після виконання цієї команди проект зазвичай доступний за адресою:
[http://localhost:5173](http://localhost:5173)

---

## 5. Базові команди

| Команда | Опис |
| :--- | :--- |
| `npm run dev` | Запуск сервера розробки |
| `npm run build` | Збірка проекту для продакшену (папка `dist`) |
| `npm run lint` | Перевірка коду на помилки за допомогою ESLint |
| `npm run preview` | Попередній перегляд збірки (`dist`) локально |

---

## 6. Поради для розробника

- **Розширення для VS Code**: Рекомендуємо встановити `ESLint` та `Prettier` для автоматичної перевірки та форматування коду.
- **Git Flow**: Перед початком роботи над новою функцією створюйте нову гілку: `git checkout -b feature/назва-фічі`.

Успіху в розробці!
